# Release Checklist

## Pre-release
- All CI checks passing.
- Security and dependency scans reviewed.
- Migration and rollback steps documented.
- Feature flags and kill switches validated.

## Release Window
- Deployment runbook assigned to on-call owner.
- Smoke tests for web, mobile, API, admin, worker completed.
- Billing and auth critical path validated.

## Post-release
- Error/latency dashboards stable.
- Conversion and onboarding metrics monitored.
- Support team informed with known issues and response template.
